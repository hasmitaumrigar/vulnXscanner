📊 Workflow Stages

🟢 Stage 1: Initiation User input form submission with target and scan type choice.
🟣 Stage 2: Validation DNS resolution validates IP format and detects IPv4 vs IPv6 automatically.
🟠 Stage 3: Setup Socket family selection and thread pool initialization based on scan type.
🔵 Stage 4: Scanning Multi-threaded port checking using socket.connect_ex() for each port.
🟤 Stage 5: Analysis Banner grabbing, service identification, and severity mapping.
💜 Stage 6: Real-time Communication WebSocket updates delivering live progress to the browser UI.
🔵 Stage 7: Persistence Results saved to JSON history with maximum 50 scans storage.
🟢 Stage 8: Completion Ready for user actions: analysis, export, or new scan.
